Q : What is Event Loop ?

ANS: Event loop ek core concept hai in JavaScript aur web browsers mein, jo asynchronous tasks ko handle karne ke liye use hota hai. JavaScript single-threaded hai, iska matlab hai ki ek waqt mein ek hi kaam kar sakta hai. Lekin jab asynchronous tasks aate hain, jaise ke I/O operations (file read/write), network requests, ya timers (setTimeout), to event loop ensure karta hai ke JavaScript apna main kaam khatam karne ke baad asynchronous tasks ko handle kare.
Call Stack : Yeh wo jagah hai jahan JavaScript ke synchronous functions ko ek ke baad ek execute kiya jata hai
Task Quee: Jab asynchronous operations complete hote hain, unke callbacks ko task queue mein daal diya jata hai. Lekin yeh tab tak execute nahi hote jab tak call stack empty na ho.
Event Loop : Event loop continuously check karta hai agar call stack empty hai ya nahi. Jab call stack empty ho jata hai, event loop task queue se pehle callback ko utha ke call stack mein daal deta hai taake wo execute ho sake.
Conclusion:
Event loop asynchronous tasks ko efficiently handle karta hai bina JavaScript ka synchronous execution block kiye.

Q : What is difference between var , const and let ?

ANS:In JavaScript, var, let, and const are used to declare variables, but they have different behaviors regarding scope, hoisting, and mutability.
var :
Scope: Function-scoped or globally-scoped. Variables declared with var are accessible throughout the entire function or globally if declared outside a function.
Hoisting: var is hoisted, meaning the declaration is moved to the top of its scope, but not the initialization. So, you can reference a var variable before its actual line of declaration, though it will be undefined until it's initialized.
Redeclaration: Variables declared with var can be redeclared in the same scope without an error.
Let :
 Scope: Block-scoped Variables declared with let are only accessible within the block (e.g., within {}) they are defined.
Hoisting: let is hoisted but cannot be used before declaration (throws a ReferenceError if accessed before initialization due to "temporal dead zone").
Redeclaration: let cannot be redeclared within the same scope, but it can be updated.
Const :
Scope: Block-scoped like let. The variable is only accessible within the block where it is defined.
Hoisting: const is also hoisted but cannot be accessed before declaration due to the "temporal dead zone."
Mutability: Variables declared with const cannot be reassigned after their initial assignment, though if the variable holds an object or array, the contents of the object or array can be modified.

Q: what is Shallow Copy and Deep Copy?

Agar object ya array ke andar nested objects ya arrays hon, toh shallow copy unki references ko copy karti hai, na ke unka actual data. Iska matlab agar aap shallow copy ka use karke ek nested object ko modify karte hain, toh original object ka nested data bhi change ho jata hai kyunki dono same reference share karte hain.
Shallow Copy Methods:
Object.assign(): Yeh shallow copy banata hai.
Spread operator (...): Yeh bhi shallow copy banata hai.
ANS: Deep Copy: Deep copy ka matlab hai ke object ya array ki har level ka data copy kiya jata hai, including nested objects and arrays. Deep copy mein ek completely independent copy banti hai, jisme koi bhi reference share nahi hota. Agar deep copy ko modify karte hain, toh original object/array bilkul unaffected rehta hai.

Q : what did do Use Effect dependencies array do if it have value or not ?

ANS:The useEffect hook in React runs side effects in function components.
No Dependencies Array: If you don't pass a second argument (i.e., no dependencies array), the effect will run after every render (on both mount and update).
Empty Dependencies Array []: If you pass an empty array, the effect will run only once, after the initial render (i.e., when the component mounts). It won't run again unless the component unmounts (in which case the cleanup function will run, if provided).
Dependencies Array with Values [dep1, dep2, ...]: When you pass an array of dependencies, the effect will run only when any of the dependencies change. React compares the current values of the dependencies with their previous values. If any value has changed, the effect will re-run.
Summary of Execution: No array: runs on every render. Empty array: runs once on mount. Array with values: runs when a dependency value changes.

Q : Life Cycle Method of components ? Mounting ,Updating Unmounting

ANS: The Life Cycle Method of components in React refers to the sequence of stages or phases that a React component goes through from its creation to its eventual removal or replacement. Mounting Phase (Component Creation): This phase occurs when a component is created and inserted into the DOM for the first time. Updating Phase (Component Re-rendering): This phase occurs when a components props or state change. Unmounting Phase (Component Removal): This phase occurs when a component is removed from the DOM.
